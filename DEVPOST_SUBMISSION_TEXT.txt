## Inspiration

Traders waste precious seconds switching between their chart, a separate
analysis tool, and a risk calculator. By the time the setup is ready,
the candle has moved. I wanted to eliminate that friction entirely — an AI
that sits on top of your screen, sees exactly what you see, and gives you
the full trade plan in seconds without touching anything else.

## What it does

NanoMech is a real-time AI trading assistant built as a transparent desktop
overlay. It uses Google Gemini 2.5 Flash's multimodal vision to "see" your
trading screen and analyze any crypto chart on any platform — no exchange
API, no DOM access needed, just pure visual understanding.

Two always-on-top overlay panels sit over your chart:

OVERLAY 1 — MARKET ANALYSIS & LIQUIDITY
Gemini visually reads the chart and returns:
• Trend — Bullish/bearish bias, moving average crossovers, market structure
• Liquidity — Order book depth, bid/ask clusters, support/resistance zones
• Momentum — Candlestick patterns, volume spikes, price velocity

OVERLAY 2 — TRADE SETUP + RISK CALCULATOR
• Entry — AI-identified entry price
• Target — Take-profit level
• Stop — Stop-loss level
• CALC (updates live as you type your Risk %):
    - Risk Amt ($) — dollar risk from your account
    - Size (Units) — position size
    - R:R Ratio — risk-to-reward

User presses SCAN NOW (or Ctrl+A+I hotkey). NanoMech captures the screen,
sends it to Gemini, parses the structured response, populates both overlays,
and runs the risk math — all in under 5 seconds. Auto Mode re-scans every
20 seconds hands-free. Every scan is logged locally with the chart screenshot
and trade data.

## How I built it

The core is a multimodal API call using the Google GenAI SDK:

    client = genai.Client(api_key=api_key)
    response = client.models.generate_content(
        model='gemini-2.5-flash',
        contents=[structured_prompt, screenshot_image]
    )

I capture the screen with mss, process the image with Pillow, and send it
to Gemini with a prompt that demands a strictly formatted response using
[ANALYSIS] and [TRADE] section tags. Regex then extracts the Entry, Target,
and Stop numeric values from the response and wires them into the live
risk calculator.

The two overlay windows are frameless Tkinter Toplevels with -topmost and
alpha transparency. A background threading model keeps the UI responsive
during the API call (with a spinning arc animation). The keyboard library
registers a global hotkey that fires even when another application is focused.

## Challenges

Prompt consistency — getting Gemini to always return prices in a parseable
numeric format required structured prompt engineering with explicit fallback
handling when the format fails.

Thread-safe UI — Tkinter is single-threaded, so all UI updates had to be
routed through root.after() callbacks from the analysis thread.

Overlay transparency — tuning alpha values and color choices to keep text
readable across both dark and light chart themes.

## Accomplishments

- Fully functional multimodal AI agent that works on ANY chart platform
  without any exchange API or DOM access
- Seamless overlay UX that never interrupts the trader's workflow
- Complete risk management calculator driven entirely by AI-extracted prices
- Clean structured outputs from Gemini through careful prompt design

## What I learned

- How to use Google GenAI SDK for multimodal image+text inputs
- Prompt engineering for structured, reliably parseable AI output
- How Gemini 2.5 Flash handles financial chart images and the level of
  visual detail it can extract (moving averages, order book data, candles)
- Building production-grade always-on-top desktop overlays with Tkinter

## What's next

- Voice output — NanoMech speaks the trade setup aloud
- Multi-monitor support — choose which screen to analyze
- Trade history tracker — log how AI setups performed over time
- Cloud version — run NanoMech as a hosted web service on Google Cloud Run

FINDINGS & LEARNINGS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
The most surprising finding was how capable Gemini 2.5 Flash is at reading
financial charts from a raw screenshot — it correctly identifies candlestick
patterns, moving average crossovers, volume spikes, and specific price levels
with no structured data input at all. This confirms the real-world viability
of UI Navigator agents that work purely from visual input, making them
platform-agnostic and far more flexible than API-dependent tools.

Prompt structure is critical. Using explicit section tags ([ANALYSIS],
[TRADE]) and demanding specific output formats enabled reliable regex parsing
of Gemini's response to extract numeric values for the live risk calculator.
